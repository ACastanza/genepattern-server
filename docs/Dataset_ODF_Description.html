Dataset ODF format

Introduction:

This document can only be understood in the context of the ODF Format documention.  If you have not done so please read it first (ODF Documentation).

The ODF format is supposed to be somewhat similar to the RES or GCT file formats for Datasets.  The main difference is in the header.  The body of data contains the expression level values for each gene in each sample. Thus the main data block (after the header lines) is a matrix of values.  The matrices columns are defined by a name and optionally a description.  The rows have a name (name of the gene for instance) and a description (description of the gene).  The columns are the expression values for each gene in a sample.  Each sample has expression values for the same named gene in a row.  So if the first gene is a particular Tyrosine Kinase then each of the samples will have expressions values for that particular Tyrosine Kinase in the first row. See the Example_Dataset.odf for reference.

Typical ODF header for a Dataset

  1. ODF 1.0
  2. HeaderLines=7
  3. Model=  Dataset
  4. COLUMN_NAMES:   Name    Description       Sample 1      Biopsy_2      Biopsy_4
  5. COLUMN_DESCRIPTIONS:    Sample from DFCI  Sample from UK   Sample from Children's
  6. COLUMN_TYPES:   String  String  float  float  float
  7. RowNamesColumn=0
  8. RowDescriptionsColumn=1
  9. DataLines= 3
Lines 1 and 2 are the required first and second lines that signify that this is an ODF formatted file and how many header lines follow before the main data block.  Line 3, required to be somewhere in the header of an ODF file, defines this ODF file as containing Dataset data.  Line 4 is the tab delimited list of names of the columns. Line 5 is the tab delimited list of column descriptions for each column. Line 6, is required for any ODF file that has a main data block, defines the type of data in each column.  Line 7 defines which column will have the row names.  Line 8 defines which column will have the row descriptions.  Line 9, is required for any ODF file that has a main data block, is the number of rows in the main data block.

Notes:

The number of data lines can be quite large. For example, the U133A chip measures the expression values for about 20,000 human genes.  A Dataset created from several samples using the U133A gene chip could have a large value for the DataLines tag.

The COLUMN_NAMES:, COLUMN_DESCRIPTIONS:, and COLUMN_TYPES: lists must have the same number of elements.  Also the number of elements must be equal to the number of columns in the main data block.

The COLUMN_NAMES:, and  COLUMN_DESCRIPTIONS: could be empty, that is with the proper number of tabs and no text.

The row names and row descriptions could be defined as ROW_NAMES:, and ROW_DESCRIPTIONS: lists instead of having columns containing the row names and descriptions.  This is like the COLUMN_NAMES:, COLUMN_DESCRIPTIONS: lists.  Either ROW_NAMES: or RowNamesColumn= tags will be used, but not both. The same goes for using either ROW_DESCRIPTIONS: or RowDescriptionsColumn= but not both.

General note about the header: It is possible that modified dataset files could exist that have additional tags not defined here.  Any parser of a dataset should not fail if this is encountered but just keep track of the unknown attributes.  Or at the very least ignore them.

Main Data Block (example first few lines)

1000_at    X60188 HSERK1 Human ERK1 mRNA    145.3   240.37823    158.66888
1001_at    X60957 HSTIEMR Human tie mRNA    20.5    31.139397    14.053186
1002_f_at  X65962 HSCP450 H.sapiens mRNA    -9.6    118.06088    -8.287777

The main data block is consistent with the header.  The first COLUMN_NAMES element is "Name".  This label is associated with the first  column (values: 1000_at, 1001_at, and 1002_f_at).  The second column's label is "Description" which is associated with the second column of the main data block.  The next three columns are floating point numbers that represent the gene expression values for each of the samples.

Notes:

The first two columns are just text data, and next three columns only contain floating point values.  This is consistent with the String, String, float, float, float elements in the COLUMN_TYPES: list.
